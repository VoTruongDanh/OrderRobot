  - add item by speech
  - review cart
  - confirm order
  - show QR/payment flow
- Admin changes for voice/listen mode should apply immediately to kiosk.
- Tax is configurable from admin and currently defaults to `10%`.

## Known Bugs & Fixes

### Bug: Minus button doesn't decrease quantity (FIXED)
**Symptom:** Clicking minus (-) button in cart sends request to backend but quantity doesn't decrease.

**Root Cause:** 
Frontend sends `"bỏ 1 [item name] size [size]"` to backend, but backend session cart may be empty (after restart or session timeout), causing the remove command to fail silently.

**Fix Applied:**
Changed minus button to update local cart directly without calling backend:
- Decrease quantity by 1 in local cart state
- Remove item if quantity becomes 0
- Update UI immediately
- Backend will be synced during checkout via rehydration logic

**Files Touched:**
- `apps/kiosk-ui/public/stitch_robot_3d_control_center.html` - Modified cart minus button handler
- `services/ai-backend/app/services/conversation_engine.py` - Added debug logging for remove operations

**Benefits:**
- Instant UI feedback (no network delay)
- No dependency on backend session state
- Consistent with optimistic update pattern used by plus button
- Backend sync happens automatically during checkout

### Bug: Second confirm doesn't show QR (FIXED)
**Symptom:** On some machines, clicking "Xác nhận lần 2" doesn't show QR code payment popup.

**Root Cause:** 
Core backend was using empty CSV menu instead of remote POS API when creating orders, even though `POS_MENU_SOURCE_MODE=remote_strict` was configured.

The issue was in `create_order()` logic:
```python
# OLD (BROKEN):
remote_live_mode = self.remote_pos_enabled and self.remote_menu_strict_enabled
if remote_live_mode:
    menu_items = self._get_remote_menu_items_by_ids(...)  # Remote API
else:
    menu_items = self.menu_repository.get_items_by_ids(...)  # CSV (empty!)
```

When `remote_pos_enabled=False` (e.g., auth not ready), it fell back to CSV even though `remote_menu_strict_enabled=True`.

**Fix Applied:**
Separated menu validation from order creation:
- `use_remote_menu` = `remote_menu_strict_enabled` (for menu validation)
- `remote_live_mode` = `remote_pos_enabled AND remote_menu_strict_enabled` (for POS order creation)

Now when `POS_MENU_SOURCE_MODE=remote_strict`, menu validation ALWAYS uses remote API, never CSV.

**Files Touched:**
- `services/core-backend/app/services/order_service.py` - Fixed create_order logic
- `services/core-backend/app/main.py` - Added logging
- `services/ai-backend/app/services/conversation_engine.py` - Added error handling

**Verification:**
- Check logs: `use_remote_menu=True` when `POS_MENU_SOURCE_MODE=remote_strict`
- Order creation should succeed with remote menu items
- QR popup should appear after second confirm

**Remaining Risks:**
- If remote API is down, order creation will fail (no CSV fallback by design)
- Need to handle remote API errors gracefully

## Important Frontend Rules

- File most frequently touched: `apps/kiosk-ui/public/stitch_robot_3d_control_center.html`
- Cart merging must be size-aware:
  - never merge by bare `item_id`
  - use a size-aware key such as `item_key = item_id + size`
- Manual UI add flow and voice optimistic add flow both must preserve separate cart lines per size.
- Do not trust backend cart sync blindly for manual/optimistic flows:
  - empty payload carts must not wipe a valid local optimistic cart
  - backend item-id-only carts can collapse different sizes if applied directly
- Exception: manual size selection coming from the voice `clarify_size` modal must allow backend cart sync, otherwise tapping a size can succeed on backend but appear unchanged on frontend.
- Voice checkout behavior:
  - first confirm opens local order review popup
  - second confirm submits actual checkout
- When backend asks `clarify_size`, kiosk should also open the in-app size picker so users can tap size manually instead of relying only on STT.
- For voice `clarify_size`, tapping a size should use the frontend manual-add path with explicit `itemId + size + unitPrice` so the cart updates immediately even if backend pending-size context is flaky.
- Review popup and mini cart are portrait-friendly and part of the expected UX.
- Size selection uses in-app modal, not `window.prompt`.
- Out-of-stock sizes must stay disabled in picker and clearly labeled.
- Chat mini panel should keep the full conversation history scrollable, but visually show about 3 recent chat blocks by default.
- Order invoice popup should support temporary hide/reopen without breaking the active order flow, and its auto-close window is now 3 minutes.

## Important Backend Rules

- File most frequently touched: `services/ai-backend/app/services/conversation_engine.py`
- Conversation should favor kiosk ordering UX over generic fallback:
  - short negative replies like `khong` after add/size flow should continue order flow, not drop to fallback
  - common STT mishearing aliases for `size` should still resolve correctly
- Local review popup does not automatically mean backend is awaiting confirmation:
  - second-step checkout from kiosk must send an explicit `quick_checkout` signal
  - otherwise backend can interpret the second confirm as only the first backend confirmation step
- Broken Vietnamese spacing / no-tone bridge text should be repaired or replaced with safe accented replies before showing on frontend.
- Single clear item match with ordering intent can auto-add even if confidence is not very high.
- Stream responses must be safe against invalid Unicode surrogate data.

## TTS / STT Notes

- TTS engine is VieNeu.
- Important compatibility lesson:
  - old VieNeu runtime + Turbo model caused slow/noisy output
  - safe fallback path was added for older runtime
- Frontend WS playback must respect backend `sample_rate`, not assume fixed `24k`.
- Current `.env.example` still shows:
  - `TTS_ENGINE=vieneu`
  - `VOICE_LISTEN_MODE=always`
  - `VOICE_ALWAYS_LISTEN=true`
  - `TTS_STREAM_PLAYBACK_RATE=1.15`
  - `CHAT_CLEAR_ON_ORDER_COMPLETE=false`

## Regressions To Avoid

- Same product in different sizes collapsing into one cart line.
- Voice confirm staying in chat instead of opening the local review popup.
- Empty backend cart payload wiping the local mini cart after manual add.
- English item names being degraded by Vietnamese alias replacement and failing explicit match.
- Garbled Vietnamese without diacritics appearing in kiosk reply text.
- Turbo VieNeu config causing abnormally slow or noisy speech on old runtime.
